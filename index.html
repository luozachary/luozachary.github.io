区别强化学习和其他类型学习的最重要的特征是，它使用训练信息来评估所采取的行动，而不是通过正确的行动来指导。

K-armed Bandit Problem

你有k个不同的行为可以选择，每次选择后都会收到一个数值奖励，你的目标就是在一定的时间周期内最大化总奖励值，如1000此选择机会。这就是k-armed bandit problem。

当分母为0时，我们定义一个默认值Q1(a)=0；当分母无限大的时候，Qt(a)将会聚合成q*(a)。因此根据我们的预估值，最简单的行为选择准则便是选择最大的估算行为奖励值。
这种贪心选择行为每次都利用现有的知识去最大化即刻奖励，并没有去探索(这也是著名的double e问题，exploit & explore)。于是我们在大多数时候都是采用贪心选择，但是存在概率ε会在所有的行为中随机挑选一个行为。

行为价值

我们令Ri表示在第i次选择之后接收到的该选择的奖励，Qn表示该行为在被选择了n-1次之后的预估价值，可以简单的写成Qn=(R1 + R2 + … + Rn-1)/(n-1)。这个计算方式需要存储所有的奖励记录，我们可以找到更有效的计算公式。

上面谈论的求平均的方法适合应用于固定的环境下，但是RL问题很多都是出于不稳定的环境下，因此需要引入一个步长因子来加重最近奖励的权重，Q(n+1) = Qn + α(Rn - Qn)，α∈(0, 1]。

  2. 查准率、查全率与F1
    在介绍这些度量之前，先来看看混淆矩阵。

    查准率(precision)和查全率(recall)分别定义为：
    P=TP/(TP+FP)
    R=TP/(TP+FN)
    查准率和查全率是一对矛盾的度量，一般来说，查准率高时，查全率往往偏低；反之亦然。在信息检索中，查准率就是检索出的信息有多少比例是用户感兴趣的；查全率就是用户感兴趣的信息有多少被检索出来。
    可根据学习器的预测结果对样例进行排序，排在前面的是学习器认为最可能是正例的样本，排在最后的则是学习器认为最不可能是正例的样本。按此顺序逐个把样本作为正例进行预测，则每次可以计算出当前的查全率、查准率，并以查准率为纵轴、查全率为横轴构造查准率-查全率曲线，简称P-R曲线。
    P-R曲线是非单调、不平滑的。P-R曲线可用来评估学习器的优劣。若一个学习器的P-R曲线被另一个学习器的P-R曲线完全包住，则后者的性能优于前者。如果两个学习器的曲线发生交叉，则通过二者面积的大小来比较，面积大的表示查全率和查准率双高比较优秀，但不太容易计算曲线（不平滑）的面积，因此通过平衡点（Break-Even Point，简称BEP）来度量。BEP是坐标上查准率等于查全率时的点，平衡点值越大，学习器越优秀。

    ERP过于简化，定义F1常量来比较学习器P-R曲线的性能：F1=2PR/P+R=2TP/(样例总数+TP-TN)
    不同的应用场合，对查全率和查准率的侧重不同，如在商品推荐中，为尽可能少打扰用户，希望推荐的内容却是用户所感兴趣的，查准率更重要；在逃犯信息检索中，希望尽可能少漏掉逃犯，此时查全率更重要。对查准率和查全率的不同偏好，可用F1度量的一般形式Fβ，定义为：Fβ=(1+β2)     P R/((β2P)+R)。
    其中β>0度量了查全率对查准率的相对重要性；β=1时就是标准的F1；β>1时偏好查全率；β<1时偏好查准率。

马尔科夫决策过程

发表于 2018-03-04 | 分类于 技术知识

Markov Processes

由上一文章强化学习简介中提到马尔科夫的必要条件，我们可以知道state拥有历史上所有的相关信息；且当state是已知的情况下，历史信息就可以抛弃。
马尔科夫过程就是一个随机的过程，比如一序列的拥有马尔科夫性质的随机状态S1, S2…

Markov Reward Process

马尔科夫奖励过程是一个带有价值的马尔科夫链。

最终获取最大奖励值为整个过程的目标，其中折扣因子表明对未来价值的重要性。折扣因子在[0,1]，折扣因子越小表示越在乎当下价值不关心未来价值。

MRP的状态价值方程v(s)将返回状态s的价值以及未来的价值，v(s) = E[Gt | St = s]。价值方程一定遵循贝尔曼方程，当前状态的value等于立刻价值加上折扣因子乘以下一状态的value。

Markov Decision Process

在MRP的基础上引入actions便是MDP。

政策π是在给定的状态下行为的分布，π(a|s) = P[At=a | St=s]。

  • 政策决定了agent的行为。
  • MDP的政策由当前的状态决定，而非历史。

从所有的政策中得到最大价值的政策的方程便是最优化价值方程，最优价值方程指出MDP中最优的表现行为，当得到最优价值方程时MDP问题同时也被解决。
对于政策的比较，在任何状态下如果政策π的价值≥π’的价值，那么就可以定义π优于π’。对于任意的MDP都存在以下条件：

  • 存在至少一个π*优于或者等于其他任意π。
  • 所有的最佳政策会得到最佳价值方程和最佳行为价值方程。

强化学习简介

发表于 2018-02-27 | 分类于 技术知识

书籍推荐

  • An Introduction to Reinforcement Learning, Sutton and Barto, 2014
  • Algorithms for Reinforcement Learning, Szepesvari, 2010

强化学习的特性

  • 没有监督者，只有奖励信号。
  • 反馈延迟，不是实时的。
  • 时间很关键。
  • agent的行为会影响其后续接收的数据。

强化学习例子

  • 飞机特技表演训练。完成一个动作不会立刻得到反馈，只有完成一次表演才能知道好与不好。
  • 管理投资。
  • 控制发电站。对于不同情况，做出不同判断，最终高效率的进行发电。

强化学习

奖励R是一个很重要的反馈信号，代表agent表现的如何，agent的目的就是最大化奖励。
例如：飞机特技做对了则+ve reward，损坏了则-ve reward；管理投资最后盈利的金额则为奖励。
为了长远的更大的奖励，当前的行为不一定是最佳的。比如管理投资，当前的投资行为可能造成亏损，但是需要考虑长期的效果。

agent通过观察环境从而做出行为，环境通过行为反馈奖励给agent，并且行为将影响环境。这个过程行程一个循环，强化学习需要得到最大奖励。
History是一系列的观察、行为和奖励。通过history，agent选择action，environment选择observations/rewards。agent的每一次判断，不可能每次读取完整的history，因此引入state，是history的函数表示St = f(Ht)。

环境状态：可以看做概括信息的数字集合，决定agent下一步采取的行动。环境状态并不都是对agent可见，agent看到的环境状态也可以是无关的信息。
agent状态：agent的内部呈现，决定下一步行为，RL算法的数据来源。
根据agent观察环境的角度可以分为两种：

  • fully observable environments: agent state = environment state = information state,其实这就是一个Markov decision process(MDP)。
  • partially observable environments: agent不是直接观察环境，agent state != environment state，称为partially observable Markov decision process(POMDP)。

强化学习代理

一个AL agent可能会包含以下一个或多个组成：

  • policy: agent behavior function，agent如何选择action。
  • value function：how goood is each state and/or action，action预期的reward。
  • model：agent’s representation of the environment，agent认为的环境，而非真正的环境。

policy就是agent的行为，是由state到action的映射：deterministic policy, a = π(s)；stochastic policy, π(a|s) = P[A = a | S = s]。
value function是对未来奖励的预测，用于评估状态的好坏，从而选择“好”的行为。

model会预测环境的变化，预测下一步的状态和奖励。

强化学习内的问题

强化学习与规划的差别：强化学习最初对环境并不知晓，代理会与环境交互，代理将提升方针；规划则清楚的知道环境并且不与环境交互。
Exploration & Exploitation：exploration探寻环境更多的信息，exploitation利用已知的信息最大化奖励，如何平衡这两点是很重要的。比如：是去一家喜欢的饭店吃饭，还是去探寻尝试另一家。
